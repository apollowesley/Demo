## 概述

API 请求失败时，服务器会返回一个 json 格式的响应体用于定位具体错误原因。如：
```
{
  "content": {
    "错误字段": [
      "错误描述1",
      "错误描述2"
    ]
  },
  "code": 400,
  "error": {
    "type": "错误类型"
  }
}
```
该响应体的字段如下：

* `code`: 错误码。具体含义请见 [API 错误码表](#api)
* `content`: 错误的具体类型。
* `error`: 错误类型。

## API 错误码表


| 错误码 | 含义   |
|-----  |---------------------------------
| 400   | 参数错误
| 401   | 认证失败
| 403   | 未经过授权
| 404   | URL不存在
| 405   | 为许可的方法
| 500   | 服务器错误
